# Decision Log — Mealie Android Brief

## 2026-05-21

- **Tech stack**: Native Android (Kotlin + Jetpack Compose). Rationale: Android-only scope, offline-first requirements best served by Room DB, share intents are first-class on native. iOS ruled out — no Mac available for compilation.
- **Auth approach**: JWT flow (username/password → access + refresh token stored in EncryptedSharedPreferences). API tokens rejected — admin-only creation makes them unsuitable for multi-user scenarios.
- **Server URL**: Configurable at setup — multiple Mealie instances exist across users.
- **MVP scope**: Login persistence + offline shopping list with check-off. Recipe offline and URL-share import are post-v1.
- **Mode**: Coaching path chosen by user.
- **QR code / setup link onboarding**: Moved to post-v1. V1 uses a simple setup dialog for server URL + credentials. Rationale: reduce v1 scope complexity.
